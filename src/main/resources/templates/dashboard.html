About

Open Sensor Data, your go-to platform for seamlessly generating and overseeing data from your sensors, all at no cost. Experience the ease of crafting and monitoring your sensor data effortlessly. Whether you're setting up sensors for your community or tapping into the collective data from public sensors, our API makes the process a breeze. Join us in unlocking the full potential of sensor technology with Open Sensor Data. To contact us, please send an email to opensensordata@gmail.com.

Tutorial

In our beginner's tutorial, we will use the LOLIN(WEMOS) D1 Pro Mini MCU (microcontroller) and the DHT22 sensor. First, we will connect our MCU to the DHT22 sensor. To do this, we need to connect the DHT22 data pin to the MCU's GPIO (General-purpose input/output) D7, also called pin 13. Next, we connect GND to GND and the DHT22's VCC pin to the MCU's 3v3 pin. Connections:
MCU pins DHT22 pins
D7 DATA
GND GND
3v3 VCC
After that, you can power your MCU in any way you want: from the battery or USB. At the moment, we are only powering our MCU by USB. We used the Arduino IDE to program the microcontroller. Go to File > Preferences... and in the "Additional boards manager URLs" field, add the URL to import the ESP8266 package:
http://arduino.esp8266.com/stable/package_esp8266com_index.json
Next, you need to download the esp8266 git repository and navigate to the certs-from-mozilla.py script to get the trusted CAs.
https://github.com/esp8266/Arduino
run script certs-from-mozilla.py
cd "{ your-path }\Arduino-master\libraries\ESP8266WiFi\examples\BearSSL_CertStore" python certs-from-mozilla.py
Get the folder with the generated data and add it to your Arduino sketch folder. Next we need to generate your certificate with tool cert.py:
cd "{ your-path }\Arduino-master\tools" python cert.py -s open-sensor-data.systems
Import generated certs.h to sketch folder (Sketch > Add File...). certs.h example:
Now copy and paste our code to get data and upload to our service. Dont forget to change Wifi credentials, sensor id and your key. Firstly, our device connects to wifi and sends a message to our service that the device is connected, and also initialises the the LED pin. In the loop method, if our device has been connected to Wi-Fi, we turn on the LED and read the humidity and temperature from the sensor. In the loop method, we read the humidity and temperature from the sensor and send this data to the service.
Upload the Arduino code to the MCU and check for any exceptions in the Serial Monitor (Tools > Serial Monitor and select the 115200 baud option). If your data is displaying in our website you completed this tutorial. Well Done! If you have any questions or suggestions, please contact opensensordata@gmail.com.
